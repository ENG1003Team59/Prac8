Help

☀ Launch Page


☑ Initial page for users, should contain some sort of link to the next page or features / pages of the app

☀ Record Page


☑ Start recording route function. This function will start recording the route.
☑ Stop recording route function. This functions stops recording, if it was recording a route.
☑ Clear recorded route function. This function clears the recorded route and resets the record page to a clean slate.
☑ Save recorded route function. This function saves the recorded route to the device’s local storage.
☑ While recording, the route should be clearly displayed on the screen to the user using either lines or breadcrumbs (dots).
☑ The page’s state should be visually represented (i.e. recording or not recording)
☑ The current location and accuracy of that estimated location should be also always shown to the user.
☑ The starting point and ending point of the route should be clearly labelled, even while recording (while recording just the starting point).
☑ When saving the route, the user should be able to enter a name for the route to be saved.

☀ View Page


☑ The user should be able to select a route to display
☑ The route should be shown on the screen with the start and end points labelled.
☑ Only one selected route should be shown at any time.
☑ The screen should also show the average speed, total distance and total time taken for the selected route.
☑ The user should be able to delete a route
☑ If there are no existing routes, then the delete function should not be enabled
